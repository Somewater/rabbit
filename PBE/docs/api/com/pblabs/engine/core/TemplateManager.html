Classpublic class TemplateManager
InheritanceTemplateManager flash.events.EventDispatcher

The template manager loads and unloads level files and stores information about their contents. The Serializer is used to deserialize object descriptions.

A level file can contain templates, entities, and groups. A template describes an entity that will be instantiated several times, like a bullet. Templates are left unnamed when they are instantiated.

An entity describes a complete entity that is only instantiated once, like a background tilemap. Entities are named based on the name of the xml data that describes it.

A group contains references to templates, entities, and other groups that should be instantiated when the group is instantiated.

loadFile()method 
public function loadFile(filename:String):void

Loads a level file and adds its contents to the template manager. This does not instantiate any of the objects in the file, it merely loads them for future instantiation.

When the load completes, the LOADED_EVENT will be dispatched. If the load fails, the FAILED_EVENT will be dispatched.

Parameters
filename:String — The file to load.
registerEntityCallback()method 
public function registerEntityCallback(name:String, callback:Function):void

Register a callback-powered entity with the TemplateManager. Instead of parsing and returning an entity based on XML, this lets you directly create the entity from a function you specify. Generally, we recommend using XML for entity definitions, but this can be useful for reducing external dependencies, or providing special functionality (for instance, a single name that returns several possible entities based on chance).

Parameters
name:String — Name of the entity.
 
callback:Function — A function which takes no arguments and returns an IEntity.
