changeAnimationEventproperty 
public var changeAnimationEvent:String

If set, the name of the event to listen to for animation changes on the owning entity. This is not required, but offers greater performance than checking for the name of the animation on every frame. No data is read from this event, it is simply a signal to check the currentAnimationReference.

Property indicating the time at which the current animation started (in virtual time ms). This can be used to make sure that animations don't "lag" behind their actions if framerate is low.
